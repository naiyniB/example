# 二、已经完成的学习内容

目前已经理解：

### 1 LLM 的基本工作方式

LLM 本质是：

```
输入 tokens
↓
预测下一个 token
↓
生成文本
```

所以：

- Prompt 是 **概率引导**
- 不是 **强制规则**

---

### 2 Agent 的核心结构

已经理解：

```
Agent = LLM + Executor
```

其中：

**LLM**

负责：

```
思考
决策
规划
```

**Executor（执行器）**

负责：

```
执行工具
调用 API
管理循环
管理 memory
```

---

### 3 Agent Loop

Agent 的核心执行结构：

```
User Input
     ↓
LLM 思考
     ↓
决定调用工具
     ↓
Executor 执行
     ↓
结果返回 LLM
     ↓
LLM 再思考
     ↓
直到任务完成
```

也就是：

```
Think → Act → Observe → Think
```

---

### 4 ReAct Pattern

已经理解：

```
ReAct = Reason + Act
```

LLM 每一步：

```
Reason（思考）
↓
Act（调用工具）
↓
Observe（看到结果）
↓
Reason
```

---

### 5 Prompt Tool Agent（当前实现）

用户已经写出一个 **基础 Agent**：

LLM 被 prompt 约束返回：

```json
{
 "tool": "getWeather",
 "args": {...},
 "next": "继续"
}
```

Agent：

```
解析 JSON
↓
调用工具
↓
保存 memory
↓
继续 loop
```

示例工具：

```
getWeather(city)
searchNews(topic)
```

---

### 6 Memory 机制

用户已经理解：

Agent 每一步会记录：

```
memory = [
 {tool,args,result}
]
```

然后在下一轮传回 LLM：

```
LLM 能看到之前做了什么
```

---

### 7 为什么 LLM 可能不遵守 JSON

已经理解：

因为：

```
LLM 是文本生成
不是程序执行
```

所以可能：

- 输出非 JSON
- 输出额外文本
- 直接回答
- invent tool

因此需要：

```
Guardrails
Error handling
Retry
```

# 三、重要概念澄清

用户已经理解以下关键区别：

### Prompt JSON Tool

当前方式：

```
Prompt 写工具
LLM 返回 JSON
Agent 解析
```

特点：

```
简单
不稳定
```

---

### Function Calling / Tool Calling

真正的方式：

```
tools schema 通过 API 提供
LLM 返回 tool_calls
Agent 执行
```

特点：

```
结构化
稳定
工业标准
```

const tools = { getWeather, searchNews }

async function runAgent(userInput) {

 const memory = []
 let finished = false

 while (!finished) {

   const response = await openai.chat.completions.create({
     model,
     messages:[
       system prompt
       user input
       memory
     ]
   })

   const llmOutput = JSON.parse(...)

   const result = await tools[llmOutput.tool](...)

   memory.push({tool,args,result})

   finished = llmOutput.next === "完成"
 }

 final summarize
}
